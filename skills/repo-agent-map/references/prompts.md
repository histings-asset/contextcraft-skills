# repo-agent-map prompts

## 1. Read-only audit

```text
请帮我生成本项目的 AGENTS.md，但现在不要写文件。

第一轮只读审计：
1. 读 README、package.json、pyproject.toml、Makefile、CI 配置、现有 .md。
2. 找出真实的启动、测试、构建命令。
3. 找出主要目录和入口文件。
4. 找出危险区域、废弃入口、生成文件。
5. 标记不确定项，不要编造。

输出一份“候选规则清单”，每条规则后注明证据来源（哪个文件哪一行）。
```

## 2. Draft short version

```text
基于刚才的候选规则，生成 AGENTS.md 草稿。

要求：
- 200 行以内。
- 只保留会改变 agent 行为的内容。
- 命令必须来自真实文件，不允许“应该可以”。
- 路径必须真实存在。
- 不复制 README 正文。
- 不写“高质量代码”“最佳实践”这类口号。
- 如果没有测试命令，写“未发现测试命令”，不要编造。
```

## 3. Reverse audit

```text
现在审计你刚写的 AGENTS.md。

逐条检查：
1. 这条是否有真实证据？
2. 这条是否会改变 agent 行为？
3. 是否过期、重复、泛泛而谈？
4. 是否应该放到独立流程文件、README、CI 或当前任务里，而不是 AGENTS.md？

输出每条规则的处理建议：保留 / 删除 / 移到别处。
```
